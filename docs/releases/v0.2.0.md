# LIA Trust Kernel v0.2.0

`v0.2.0` is the first release containing the completed V2 trust-boundary work. The project remains
pre-1.0; this is a minor-version release, not a compatibility claim of SemVer `2.0.0`.

## Highlights

- Explicit policy-bounded in-root recursive cleanup while filesystem/home/root boundaries remain
  hard denials.
- Production-path evidence for completion, AST, taint, grounding and sycophancy gates.
- Durable journal lifecycle, denial telemetry, bounded process timeouts and verified-denial memoization.
- Gemini CLI and Cursor adapters plus typed process-contract validation.
- Optional Linux namespace/Landlock confinement with narrowly scoped IP-egress/path-write assurance
  and one-shot credential delivery.
- Strict workspace proof gates and a two-runner IS-5 workflow definition.
- Public installer now pins this release, verifies `SHA256SUMS`, and rejects tampered or malformed
  artifacts without silently falling back.

## Published artifact boundary

- Verified release target: `x86_64-unknown-linux-gnu`.
- Asset: `lia-v0.2.0-x86_64-unknown-linux-gnu.tar.gz` containing one executable named `lia`.
- Digest manifest: `SHA256SUMS`.
- Other platforms build the `v0.2.0` tag from source; no prebuilt verification is claimed for them.

## Honest limitations

Hook/MCP adapters still gate only calls they observe. Complete mediation is not claimed. The optional
Linux backend confines only the attested wrapped process and retains the residual boundaries described
in `docs/threat-model.md`. Recorded/local adapter evidence is not a live OAuth-agent measurement.
Hosted GitHub Actions execution is not claimed for this release while the repository account prevents
jobs from starting; local verification and the independently downloaded public artifact are reported
separately.
