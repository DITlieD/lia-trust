# LIA Trust Kernel v0.3.0

**Multi-harness mediation depth:** Grok-class envelopes, install doctor, optional
spawn gating, and clearer subagent visibility — still GATE, not complete mediation.

## Highlights

- **Shared envelope normalize** for Claude snake_case, Grok camelCase, and sibling
  tool-name aliases (`run_terminal_command`, `read_file`, `search_replace`, …).
- **`ADAPTER_PARSE`** operator signal distinct from `FS_*` / `SHELL_*` policy denials.
- **`lia doctor`**: binary/hooks/roots/envelope self-test; exits non-zero on error checks.
- **`lia status`**: lists mediated vs known-unmediated tools for the install profile.
- **Install**: `--union-roots`; reinstall still preserves broader roots by default (0.2.2+).
- **Spawn GATE**: `Task` / `Agent` / `spawn_subagent` map to `spawn-agent` with
  `spawn_policy.allow` (default allow + signed journal). Reasons `SPAWN_ALLOWED` /
  `SPAWN_DENIED`.
- **Parent/child linkage** when wire provides session/parent/agent ids (journal detail).
- **Grok** is a first-class row in `docs/harness-compatibility.md` (Claude-compat path).

## Honest limitations

- Full **subagent PREVENT** is **not** claimed. Child PreToolUse is unknown/partial per
  harness; probe key `subagent_visibility` stays false without measured proof.
- Matcher profiles `broad` / `strict-mcp` and MCP mutate allowlists are **DEFERRED**
  (Ledger C). Default matcher expanded only for Task/Agent spawn.
- Complete mediation / process supervisor remains out of scope.
- Hosted IS-5 two-runner and live OAuth PREVENT remain external / non-blocking if
  billing or credentials block them.

## Artifact

- `lia-v0.3.0-x86_64-unknown-linux-gnu.tar.gz` + `SHA256SUMS`
- Installer pin: `VERSION_HINT=0.3.0`

## Upgrade

```bash
# from release assets or source
./install.sh   # or cargo install path; lia --version → lia 0.3.0
lia doctor
```

Prior tags `v0.1.0` … `v0.2.2` are preserved.
